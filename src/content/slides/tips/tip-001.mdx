_Source section: CLAUDE.md Configuration_
- **What goes in:** Non-negotiable standards only — architecture constraints, execution rules, domain conventions Claude can't infer. Keep it at ~300 lines / 2.5k tokens. Boris Cherny (Claude Code's creator) keeps his at ~2,500 tokens.
- **NOT linting/formatting:** Linters and formatters are faster, cheaper, and 100% reliable. Run them in your normal workflow (or hooks). Claude is not a linter.
- **Keep it alive:** When Claude makes a recurring mistake, say: _"Update the rules file so we never do this again."_ Review the change like code. This compounds team knowledge over time.
- **Decision matrix:** Always → CLAUDE.md · Sometimes → Skills · Isolated → Sub-agents · Automatic → Hooks · External data → MCP.

```markdown
# CLAUDE.md

## Working Agreement
- Keep changes small and focused.
- Run `npm run lint` before finalizing.
- Prefer editing existing files over adding new abstractions.

## Communication
- Explain trade-offs briefly before big changes.
```
