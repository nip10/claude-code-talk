<TipSlide
  title="Workflow: How I Actually Work"
  description="From ticket context to verified UI changes."
  tags={["Workflow", "Process", "Front-end"]}
>

1. Start from the ticket (or brief) and pull only the technical context I need.
2. Brainstorm constraints, risks, and what success looks like.
3. Write a short plan before touching code.
4. Implement in small chunks with frequent check points.
5. Verify in the browser, then verify in tests and console logs.
6. Finish with clean PR notes, trade-offs, and follow-ups.

</TipSlide>
