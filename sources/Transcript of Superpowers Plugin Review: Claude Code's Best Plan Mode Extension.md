I procrastinated a feature for a year. An automated content pipeline for my React libraries website. Fetch newsletters, extract trending libraries, research them, write tutorial articles, generate images and publish. I tried building it once with Task Master AI and older models. That attempt let me down at dead end. Manually parsing newsletter HTML with regex instead of using AI for extraction. I gave up. Then I installed the Superpowers plugin for Claude Code. Three hours later, the pipeline was working. This isn't a hype video. I'm going to show you exactly what Superpowers does, how the workflow works, and where it falls short. Because like any spec-driven methodologies, it does fall short in a few places. Superpowers is an open source Claude Code plugin created by Jesse Vincent. It has over 58,000 stars on GitHub and was officially accepted into Anthropic Plugin Marketplace on January 15th, 2026. It's MIT licensed, completely free. If you watched my previous videos of spec-driven frameworks BMAD, Spec Kit, Open Spec, Agent OS and Task Master – you know the pattern. They all try to enforce structure. Write a specification first, then implement. The problem is that most of those tools were running on older, less capable models with older tooling. And Claude Code itself has evolved. Hooks and sub-agents have been around for a while, but Skills and the Task List are new, and that combination changes what's possible. Superpowers is different. It's built directly on Claude Code's native features. It doesn't fight the platform, it extends it. Worth noting: Superpowers isn't limited to Claude Code. According to their README, it also works on Codex and Open Code. The core idea is simple enforce a senior engineer's methodology. Brainstorm first. Plan second. Execute third. With mandatory TDD at every step. Think of it as giving Claude as strict playbook that it must follow. No freestyling allowed. Two ways to install it. The marketplace method. Or you can use the “Discover” tab in the plugin menu and search for Superpowers. Either way, you need to restart Claude Code after installation. Verify it worked by running /help and navigating to the “custom-commands” tab. You should see /superpowers:brainstorm, /superpowers:write-plan and /superpowers:execute-plan One heads-up. Some people report flaky installation due to marketplace name collisions. An upstream Claude Code issue, not a Superpowers problem. I installed from the “Discover” tab and it went smoothly. The entire methodology follows three phases: Brainstorm, Plan. Execute. Brainstorming, Run /superpowers:brainstorm and describe what you want to build. Instead of jump into code, Claude enters a Socratic questioning mode. It asks detailed, probing questions and they are good questions. In my case, what AI provider for article writing? Claude Code agent via max subscription, no API key. Image generation? Open AI API. How to handle libraries that already have articles? Skip if updated less than six months ago. Flag for review otherwise. Should articles go straight to production? No. Publish to a local database first, review, then batch to production. Research depth? Deep research, not just GitHub README plus NPM. Single master script or independent skills? Master orchestrator plus independent skills. It's asking much more and better questions than Plan Mode. And there is a hard gate. It won't let you skip the implementation until the design is complete. Git worktree. After you approved that design, Superpowers automatically creates a git worktree – an isolated workspace on a new branch. Your main branch stays clean. Planning /superpowers:write-plan breaks the design into focused tasks, each with around five steps, small enough for a single agent to execute without context overflow. Mine generated 12 tasks in the Task List. After the brainstorming phase alone, 43% of context was used, which matters for what comes next. Superpowers presented two execution options: sub-agent driven in the same session, or a parallel session on the separate branch. I chose same session since I had over half the context remaining. Execution. /superpower:execute-plan dispatches a fresh sub agent per task with clean context. Only the specific task context is loaded. After each task, a two-stage review: spec compliance then code quality. Tasks run sequentially, not in parallel. Every task follows RED-GREEN-REFACTOR. Write a failing test. Make it pass. Refactor it. If Claude writes code before tests, the skill makes it start over. Finishing When all tasks complete, you get options: merge, create a PR, keep working, or discard. If you merge or discard, it cleans up the worktree. ReactLibs.dev is an Astro website I built that hosts React library tutorials. Yes, it's AI-generated content, what people now call AI slop. I built it to get hands-on practice with AI APIs and programmatic image generation, and I learned a lot in the process. I was also curious whether real people or AI agents would actually end up visiting it. The feature I'd been putting off was an automated content pipeline. Fetch React newsletter issues from the web, extract trending libraries, research the top five, write tutorial articles, generate images and publish. I tried this once before with Task Master AI. That attempt failed. It guided me toward parsing newsletters with regex instead of using AI for extraction, and tried subscribing to email newsletters instead of simply fetching the web URLs. Dead end. Superpowers ask the right questions during brainstorming, and led me to a better architecture. Use Claude itself for parsing newsletter HTML, fetch from web URLs, and structure everything as independent skills with a master orchestrator. All 12 tasks completed on the first execution pass. Nine skills created, tests passing. Sounds great, but then I actually ran the pipeline. Only two steps completed before the context hits 45%. The orchestrator was doing everything inline instead of delegating to sub agents, I had to restructure it to spin up a fresh sub agent per skill. That helped. But context still grew fast. The sub agents were outputting full HTML content back to the parent, entire newsletter bodies, full article text. I had to fix every agent to return only file paths to their saved artifacts, not the content itself. Then the real debugging started. Newsletter fetcing broke for some sources. URL pattern issues that needed source-by-source fixes. The production publish step failed because the database token didn't have the right permissions. Image generation needed specific sequencing – deploy images to the public folder first, then insert articles into the database. Otherwise, you get broken images on the live site. Multiple pipeline reruns with incremental fixes. Each run revealed the next issue. Eventually, 62 libraries found across all newsletters. Picked the top five. Five research agents running in parallel. One fun moment at the end: the “verification before completion” skill interpreted its job a bit too creatively. It ran the pipeline clean up and wiped all the artifacts I just generated. Had to rerun the final steps. Total time – about three hours. A feature I procrastinated for a year. When to use Superpowers versus Plan Mode? This is the practical question. Here's how I think about it. Plan Mode is for small to medium tasks. Quick features, bug fixes, refactors where you know roughly what needs to happen. Superpowers is for medium to large tasks. Multi-file features new subsystems, anything where the planning phase genuinely matters. For very large features expect an iterative loop regardless of what tool you use. No framework will one-shot a complex system. The good news – you're not locked in. If you're mid-project with Superpowers and hit the small or medium fix, just prompt directly or drop to regular Plan Mode. Fix it. Come back. The workflows coexist. And you don't need to memorize any documentation. Just run /superpowers:brainstorm and it guides you through the entire process interactively. Token consumption. The biggest issue. My pipeline burned through tokens because everything ran on the same expensive model. Most steps don't need Opus. Fetching HTML? Haiku. Extracting library names? Haiku. Research? Sonnet. Only article-writing truly needs the best model. You can technically set the “model” field an agent frontmatter, but neither Superpowers nor Claude Code will optimize this for you. When you ask it you to create a sub agent, it just creates one on the same model. You have to think about what level of reasoning each step actually needs and configure that yourself. No parallel execution. Tasks run sequentially even when they are independent. TDD enforcement is opinionated. If your project doesn't use TDD, the mandatory cycle creates friction with no easy way to disable it. Overhead for small tasks. Brainstorm, Plan, Execute is overkill for quick fixes. Just use Plan Mode. The planning phase is the strongest part. It asks better questions than Plan Mode, produced a design that matched my vision, and refined it in ways I hadn't considered. That brainstorming alone is worth trying the plugin. Superpowers isn't groundbreakingly different at the surface. Brainstorm. Plan. Execute. The difference is it's built on Claude Code's native infrastructure. Skills, hooks, sub agents – used the way they were designed. It's the best “Plan Mode Extender” available right now, and being in Anthropic’s official marketplace gives it staying power. It won't one-shot huge features, but it gets you to a working result through structured iteration. Three hours for a year-old backlog item is a win. What I'd want next – model tiering. Cheaper models for routine sub agent steps. Expensive models only where it matters. Though I suspect you could fix that with a simple instruction in CLAUDE.md.
