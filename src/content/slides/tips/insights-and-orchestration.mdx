<TipSlide
  title={"Insights, Orchestration, and Debugging"}
  description={"Audit past sessions with insights, use Claude as a general-purpose orchestrator, and leverage autonomous debug cycles."}
  tags={["Workflow", "Advanced"]}
>

- **`claude insights`:** Run `claude insights` to analyze past sessions and identify patterns, mistakes, and improvements. Copy actionable insights into CLAUDE.md to prevent recurring mistakes.
- **General-purpose orchestrator:** Claude Code can drive Slack, BigQuery, Sentry, Notion, and other non-coding tools via MCP and CLIs. Treating it only as a coding assistant undersells its capability.
- **Autonomous debugging:** For hard-to-reproduce bugs, ask Claude to: add debug logs, run the app, trigger actions, tail logs, and analyze. Claude orchestrates the entire debug cycle.

</TipSlide>
