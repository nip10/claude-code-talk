<TipSlide
  title={"Prompting and Project Documentation"}
  description={"Use explicit prompts, provide structured project docs, and write user stories before implementation."}
  tags={["Workflow", "Prompt Technique"]}
>

- **Explicit prompts:** State constraints, desired approach, and tool expectations explicitly. Include relevant stack/context details and direct instructions in the initial prompt. Prompt precision drives output quality.
- **Structured project docs:** Generate documentation files — PRD, architecture.md, decision.md, and feature.json (with completion criteria and a `passes` key) — to give Claude the right structured context.
- **User stories first:** Prompt Claude to write user stories covering all interaction paths before coding. Use them as acceptance criteria. After implementation, have Claude work through them one by one.

</TipSlide>
