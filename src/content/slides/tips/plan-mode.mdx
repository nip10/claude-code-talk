<TipSlide
  title={"Plan Mode: Think Before You Execute"}
  description={"Start non-trivial work in Plan Mode to pressure-test assumptions and edit the plan before execution."}
  tags={["Workflow", "Prompt Technique"]}
>

- **Plan first:** Begin non-trivial work in Plan Mode. Most rework comes from bad assumptions, not slow coding.
- **Challenge assumptions:** Start with a high-level goal, answer clarifying questions, and challenge weak assumptions before committing to an approach.
- **Edit the plan:** Treat the proposed plan as a draft. Review steps, remove vague actions, ask for alternatives.
- **Execute only when ready:** Switch to execution only when the plan is explicit, testable, and stable. Correcting plan defects is far cheaper than fixing implementation defects.

</TipSlide>
