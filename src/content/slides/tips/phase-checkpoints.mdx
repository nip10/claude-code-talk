<TipSlide
  title={"Persist Phase Checkpoints in Markdown"}
  description={"Keep lightweight phase files with current plan, task checklist, and QA gates so you can reset context without losing state."}
  tags={["Workflow", "Performance", "Advanced"]}
>

- **Phase files:** Keep lightweight phase files with current plan, task checklist, and QA gates for each feature stream.
- **Aggressive context resets:** You can reset context aggressively without losing execution state or acceptance criteria because checkpoints persist on disk.
- **Update at boundaries:** Maintain one short markdown checkpoint per active feature and update at phase boundaries (plan complete, implementation complete, verification complete).
